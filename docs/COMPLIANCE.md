# 数据合规与运维（P2 基线）

本文档描述 KyInfo 在对外运营时可作为起点的**技术与流程基线**；具体法律结论需产品/法务确认。

## 日志与保留

- **应用日志**：避免记录密码、完整 JWT、支付信息等敏感字段；错误日志可对邮箱等做掩码。
- **审计日志**：见 `AuditLogs` 表（管理操作），建议设定保留周期（如 180 天～1 年），到期归档或删除策略由运维执行。

## 用户权利（规划）

以下能力可按监管与客户要求分阶段上线：

| 能力 | 说明 |
|------|------|
| **账号注销** | 逻辑删除或匿名化用户标识；处理关联成绩/业务数据归属（删除或保留统计匿名化）。 |
| **个人信息导出** | 导出该用户账号字段及由其提交的业务数据（如成绩）为机器可读格式。 |
| **隐私政策入口** | 前端注册/登录页提供政策链接（内容由法务提供）。 |

## 当前代码状态

- 服务端已对 Root 账号资料修改做限制；审计记录关键管理操作。
- **账号注销 / 数据导出 API** 可作为后续独立需求实现；实施前需完成数据模型与法务评审。
